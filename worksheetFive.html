Concept of Prototyping & Why we Prototype

Prototyping is a non-functional mock up model that does not necessarily represent all aspects of the final application design but acts as a good representation that provides functionality of tasks and can have those functionalities tested for user friendly understanding. Prototyping is cheap and can easily make changes before implication. The purpose of prototyping is to demonstrate design options, layout, and concepts. With the design the prototype can validate requirements, explore solutions, validate that the UI is usable, help identify if proposed structure fits users work model and understand usability studies.

Significance in Development Process & Some Key Benefits

The significance of prototyping in the development process allows for proof of concept to be tested by a user to help the design team understand what the user is thinking. The functionality and desirability of tasks is tested for clear and concise communication between user and application.

Benefits

- Improve system usability and design quality
Feedback from prototypes helps to redesign the system to be better understood by users' first glance and overall improve the quality by making the design more user friendly. This involves keeping the design less cluttered, not overwhelming the user with too many functions but while having all desirable functions easily locatable.

- User real needs are considered in the design process
The needs of the user are clearly expressed in the prototype phase of the development process, these needs can be easily implemented into the design.

- Reduce development effort
Making changes in the prototype phase saves a lot of money and time as making changes later into the development can change the whole structure and may cause a lot of bugs and errors.

Basic Steps

- Choose an important user
- Determine some typical tasks
- Make screenshots or sketches with paper
- Perform a usability test
